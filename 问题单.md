问题现状                                                                                                                                                                                                                                                                              
  1. 问题现象                                                                                                                                                                                                                                                                           
  当用户通过 aaw-workflow 启动工作流并调用 sr-design skill 时，question-tracker MCP 工具调用失败，错误信息：                                                                                                                                                                            
  SessionNotFoundError: 未找到 .sdd/.current_session 会话标记文件                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 2. 根本原因

  在 commit 467aec4 "CLI化工作流" 中，aaw-workflow/SKILL.md 被完全重写：

  ┌──────────┬──────────────────────────────────┬─────────────────────────┐
  │  对比项  │          原来 (297e503)          │    重写后 (467aec4)     │
  ├──────────┼──────────────────────────────────┼─────────────────────────┤
  │ 架构     │ 步骤式手动工作流                 │ 配置驱动的 CLI 工作流   │
  ├──────────┼──────────────────────────────────┼─────────────────────────┤
  │ 状态文件 │ .sdd/.current_session            │ .sdd/<SR>/workflow.yaml │
  ├──────────┼──────────────────────────────────┼─────────────────────────┤
  │ 会话标记 │ ✅ 步骤 4a 写入 .current_session │ ❌ 被删除               │
  └──────────┴──────────────────────────────────┴─────────────────────────┘

  删除的步骤 4a 原意：
  4a. 写入会话隔离标记：
      - 将路径 `./.sdd/{SR编号}/` 写入 `./.sdd/.current_session`
      - 确保 question-tracker MCP 能正确定位状态文件

  3. 影响链条

  用户启动 aaw-workflow
      ↓
  调用 sr-design skill
      ↓
  sr-design 调用 question-tracker MCP (add_questions/get_status 等)
      ↓
  MCP 读取 .sdd/.current_session 失败
      ↓
  SessionNotFoundError → MCP 工具不可用

  4. 涉及的文件

  ┌─────────────────────────────────────┬─────────────────────────────────────────────┐
  │                文件                 │                    角色                     │
  ├─────────────────────────────────────┼─────────────────────────────────────────────┤
  │ skills/aaw-workflow/SKILL.md        │ 入口技能，缺少写入 .current_session 的逻辑  │
  ├─────────────────────────────────────┼─────────────────────────────────────────────┤
  │ skills/question-tracker-mcp/main.go │ 读取 .sdd/.current_session 获取状态文件路径 │
  ├─────────────────────────────────────┼─────────────────────────────────────────────┤
  │ skills/sr-design/SKILL.md           │ 依赖 MCP 工具，触发时机在被加载时           │
  └─────────────────────────────────────┴─────────────────────────────────────────────┘

  5. 修复方案

  在 aaw-workflow/SKILL.md 第 85 行（加载 skill 子技能前）添加：

  # 写入 .current_session（内容格式：./.sdd/{SR}/）
  printf "./.sdd/%s/" "$SR" > .sdd/.current_session
